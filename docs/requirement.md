# 🏁 [필독] "스탠다드반, 나한테 맞는 걸까?"

"스프링이라는 거대한 트랙에 서기 전, 본인이 '스탠다드' 트랙에 어울리는지 딱 정해줄게요. 너무 버거워도, 너무 쉬워도 안 되니까요!"

### ✅ 이런 분들은 스탠다드반이 취향입니다. (수강 권장)

- **"자바는 뗐는데 스프링은... 벌써 겁나요"**: 객체지향은 어느정도 이제 익숙해졌는데, 당장 내일부터 마주할 `@RestController`, `@Service` 같은 스프링 용어들이 외계어처럼 들릴까 봐 미리 걱정되는 분.
- **"Git은 써봤지만 '협업'은 처음인 분"**: 혼자서 `add`, `commit`, `push`는 해봤습니다. 그런데 팀원들과 브랜치를 나누고, 코드를 합치고(Merge), 충돌(Conflict)을 해결하는 '실전 워크플로우'는 한 번도 안 해봐서 막막한 분.
- **"JPA가 마법처럼 느껴질까 봐 불안한 분"**: 코드를 따라치면 DB에 저장된다는 건 알겠는데, 정작 그 밑단에서 SQL이 어떻게 날아가는지 원리를 모른 채 '마법'에만 의존하게 될까 봐 두려운 분.
- **"독립 선언이 필요한 분"**: 필수 과제는 어떻게든 따라 하겠는데, '도전 과제' 문구만 보면 나도 모르게 창을 닫고 싶어지는 분.
- **"에러 메시지가 무서운 분"**: 인텔리제이 하단에 빨간 글씨가 뜨면 심장부터 뛰고, 구글링이나 AI에게만 의존하게 될까 봐 내심 걱정되는 분.

### 🛑 '베이직반'에서 기초를 더 다져야 하는 분 (수강 비권장)

"아직 자바가 조금 불안하다면, 베이직반에서 근육을 먼저 키우는 게 훨씬 빠른 길입니다!"

- **"자바 문법과 여전히 내외하는 중인 분"**: 변수 선언, 반복문, 조건문을 작성할 때마다 구글링이 필수이거나, 자바 문법 에러를 해결하는 데에만 하루 시간의 대부분을 쓰고 계신 분.
- **"객체지향의 개념이 안개 속에 있는 분"**: 클래스, 상속, 인터페이스라는 단어는 귀아프게 들어왔지만, 막상 "이걸 왜 써야 하죠?"라는 질문에 답하기 어렵거나 코드로 구현하는 게 막막한 분.
- **"스프링은 '자바'라는 엔진으로 돌아가는 차입니다"**: 엔진(자바 기초)이 불안정하면 스프링이라는 차는 출발조차 하기 어렵습니다. 특히 스프링의 핵심인 DI(의존성 주입)는 객체지향에 대한 이해가 필수입니다.
- **"후퇴가 아니라 전략적 도움닫기입니다!"**: 기초가 흔들리는 상태에서 스프링으로 넘어가면 나중에 더 큰 좌절을 겪을 수 있습니다. 탄탄한 기초를 다져서 '진짜 내 코드'를 짤 수 있게 도와주는 베이직반을 강력 추천해요!

### ⚡ '챌린지반'에서 고공전을 치러야 하는 분 (수강 비권장)

"이미 기술적 정점에 서서 아키텍처를 고민하고 계신가요? 스탠다드반은 여러분에게 너무 좁을 수 있습니다!"

- **"지속적으로 '대화'가 되는 코드를 지향하는 분":** 코딩은 누구나 할 수 있습니다. 하지만 지속가능한 개발은 컴퓨터 사고를 갖고 대화할 줄 아는 개발자만 할 수 있습니다. 클린 코드와 유지보수성이 높은 구조를 통해 **코드만으로도 의도가 전달되는 커뮤니케이션**의 가치를 체득합니다.
- **"함께 일하고 싶은 개발자가 되고 싶은 분":** 단순 개발이 아닌, 제대로 알고 코드에 녹일 수 있다는것은 개발자로써 최소한의 동료에게 신뢰를 주는 태도입니다. 현직 개발자의 인사이트로 장애 처리 경험과 실무 CS 지식을 바탕으로 **팀 전체의 생산성을 높이는 기술적 성장**의 기틀을 마련합니다.
- **"시스템 설계의 '트레이드오프'를 논하고 싶은 분":** DDD, MSA, 클린 아키텍처 등 복잡한 개념을 이론으로만 배우지 않습니다. 실무에서 마주하는 제약 사항 속에서 어떤 것을 얻고 어떤 것을 포기할지 현직 개발자의 시선으로 분석하고 인사이트를 나눕니다.
- **"대용량 트래픽의 다양한 관점을 이해하고 싶은 분":** 단순히 도구를 사용하는 법이 아니라, 네트워크와 DB 최적화 등 물리적 관점과 비동기 시스템 등 애플리케이션 관점에서 **대용량 트래픽을 처리하는 전략적 설계**를 고민해볼 수 있는 인사이트를 나누고, 대용량 트래픽에 겁먹지 않을 수 있는 자신감을 장착해봅니다.

## 🗓️ 프로젝트 커리큘럼: 3.5개월간의 실전 질주 (Sprint)

**"인스타그램 클론 워크북의 빈칸을 채우며, 나만의 프로젝트로 응용합니다."**

### 🚀 Phase 1: "마법 같은 JPA의 실체를 밝히다" (2월)

- **프로젝트**: **인스타그램 핵심 기능(Member, Post)**
- **핵심**: SQL 기초 특강, Entity 매핑, **N+1 문제 해결**
- **솔루션**: 커리큘럼에 빠져있는 Native SQL, 제가 직접 '심폐소생술' 해드립니다. 인스타그램의 복잡한 DB 테이블을 직접 설계해보고, "팔로우 기능 때문에 왜 서버가 느려지는지(N+1)"를 Fetch Join으로 직접 고쳐보며 데이터의 흐름을 장악합니다.

### 🏗️ Phase 2: "무너지지 않는 탄탄한 성벽 쌓기" (3월)

- **프로젝트**: **인스타그램 고도화(다중 이미지, 해시태그)**
- **핵심**: 계층형 아키텍처, 트랜잭션(@Transactional), 예외 처리
- **솔루션**: 게시물을 올릴 때 이미지와 해시태그가 동시에 저장되어야 하죠? 하나라도 실패하면 전부 취소되어야 합니다. 인스타그램의 '피드 게시' 시나리오를 통해 트랜잭션의 롤백을 확인하고, 어떤 에러에도 서버가 죽지 않는 '우아한 예외 처리'를 배웁니다.

### 🌐 Phase 3: "세상 밖으로, 외부 세계와의 연결" (4월)

- **프로젝트**: **인스타그램 소셜 기능(OAuth2, 배포)**
- **핵심**: 소셜 로그인(페이스북/구글), AWS & Docker 실전 배포
- **솔루션**: 인스타그램에 가장 어울리는 '페이스북 로그인'을 직접 구현하며 외부 API 연동(OAuth2)의 정석을 배웁니다. 완성된 앱을 Docker에 담아 AWS 클라우드에 실제로 배포해 보는 짜릿한 경험을 선사합니다.

### 🏆 Phase 4: "대용량 트래픽? 겁나지 않아요" (5월)

- **프로젝트**: **인스타그램 최적화(Redis, Kafka)** & 최종 프로젝트
- **핵심**: Redis 랭킹 시스템, Kafka 알림 흐름 분리, 부하 테스트(K6)
- **솔루션**: "인기 해시태그 랭킹"을 위해 Redis를 도입하고, "팔로우&좋아요 알림"을 Kafka로 분리해 처리 속도를 높입니다. 내 서버에 수백 명이 들어오는 상황을 시뮬레이션(K6)하고 인덱스 튜닝으로 성능을 개선하는, '진짜 개발자'의 결과물을 리포트로 완성합니다.

## 💡 스탠다드반만의 '비밀 병기'

1. **인스타그램 워크북 (Fill-in-the-Blanks)**: 튜터가 제공하는 인스타그램 템플릿의 `// TODO` 주석을 매주 수업에서 함께 채워갑니다. "오늘 배운 게 여기에 쓰이는구나!"를 매 시간 느낄 수 있습니다.
2. **SQL 심폐소생술**: JPA를 쓰기 전, 엑셀보다 쉬운 SQL 기초 세션을 진행합니다. DB를 모르면 백엔드 개발자가 될 수 없습니다.
3. **에러 메시지 독해법**: 빨간 글씨는 적이 아니라 '힌트'입니다. 튜터와 함께 로그를 끝까지 읽으며 스스로 문제를 해결하는 근육을 키웁니다.
4. **도전 과제 길잡이**: "도전 과제 100% 달성"은 여러분의 취업 치트키가 될 것입니다. 막히는 부분은 튜터가 힌트 요정으로 등판합니다.
